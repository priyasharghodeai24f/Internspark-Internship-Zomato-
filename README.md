# Internspark-Internship-Zomato

Zomato Data Dive: Insights & Action Plan
The Process

•Forced all dirty data into strings, scrubbed out the messy text (/5, commas), and
coerced the rest into clean numbers.
•Dropped the unreadable rows to ensure strict data accuracy.

•Built out a visual dashboard using ggplot styling, focusing on horizontal bar charts
and frequency distributions.
What the Data is Telling Us

•The "Okay" Trap: Almost everyone is rated between 3.6 and 3.9. True 4.5+
restaurants are basically unicorns.

•The Epicenters: BTM isn't just popular; it completely dwarfs every other

neighborhood in sheer restaurant volume.
•The Flavor Profile: If it isn't North Indian or Chinese, it's a minority cuisine.
Themarket is highly saturated with these two flavors.
Action Items for Alfido Tech
•Incentivize the Good Reviews: Push that 3.6 average up by offering users
account credit for leaving high-quality, photo-based reviews on 4.0+ places.

•Recruit the Niche: Since the market is drowning in North Indian food, we can
stand out by aggressively onboarding rare cuisines (Authentic Thai, Mediterranean).

•Strict Pricing Sliders: Prices are all over the place. The app needs hard UI sliders
(e.g., "Under ₹500") right on the home screen.
•Student Verification: BTM's budget-heavy data screams "college crowd."
Build a
feature for 10% student discounts via university ID verification.

•The Quality Strike System: Cheap doesn't mean bad. If a restaurant consistently
drops below a 3.0 rating, we need an automated system to pause their account to
protect app trust.
